Техническое задание: сайт-каталог колец для автомобиля


1. Цель проекта
1.1 Публичный сайт с каталогом колец (автомобильная тематика), без подбора «по марке/модели авто». Пользователь фильтрует товар, оформляет заказ (создается заказ с продавцом куда падает информация о том, что он хочет купить). 
1.2 Администратор получает уведомление на e-mail, вручную связывается с клиентом, отправляет QR для оплаты. Клиент в личном кабинете видит свои заказы и их статусы. Администратор обновляет каталог и цены/наличие через загрузку двух Excel-файлов (структура файлов будет предоставлена позже и зафиксируется отдельным приложением к ТЗ).


2. Роли и доступ
2.1 Гость: просмотр каталога, фильтры, корзина/оформление заказа — по решению заказчика (обычно оформление только после регистрации или входа).
2.2 Клиент: регистрация и вход по e-mail и паролю, восстановление пароля (письмо со ссылкой сброса). Оформление заказа и чат с продавцом (админом).
2.3 Администратор: вход в защищённую админ-зону; загрузка Excel для каталога; загрузка Excel для наличия и цен; просмотр заказов(чатов с клиентами); контактные данные клиента для связи.


3. Каталог и фильтрация
3.1 Отображение списка товаров с карточками (название/артикул, ключевые характеристики, цена, наличие — по данным из системы).
3.2 Фильтры по согласованному с заказчиком набору полей (без привязки к конкретному автомобилю): например диаметр, ширина, тип/серия, бренд, ценовой диапазон, «только в наличии» — точный перечень полей определяется после анализа Excel каталога.
3.3 Сортировка (например по цене, по наличию, по новизне) — по согласованию.
3.4 Страница товара: описание, характеристики, фото (если есть в данных или загружаются отдельно — уточнить у заказчика).


4. Заказ и оплата
4.1 Оформление заказа из корзины: контактные данные (минимум необходимый для связи и исполнения заказа — согласовать: телефон, ФИО, город/доставка и т.д.), комментарий к заказу.
4.2 Оплата только вне сайта по QR-коду, который администратор отправляет клиенту удобным каналом. На сайте отображаются статусы вроде «ожидает оплаты», «оплата получена», «отправлен/готов к выдаче» и т.п. — перечень статусов согласовать с заказчиком.
4.3 После создания заказа: письмо на e-mail администратора с кратким содержанием заказа (номер, состав, контакты клиента, время). Опционально — письмо клиенту с подтверждением и номером заказа.


5. Личный кабинет клиента
5.1 Список заказов с датой, суммой, статусом.
5.2 Карточка заказа: состав, суммы, статус, история статусов (желательно с датой/временем).
5.3 Редактирование профиля: e-mail (с подтверждением при смене — по согласованию), смена пароля, телефон и прочие поля — в рамках минимально необходимого.


6. Администрирование и импорт Excel
6.1 Файл 1 — каталог: создание/обновление номенклатуры и атрибутов для фильтров и карточек (правила: полная замена vs дифф, обработка удалённых позиций — зафиксировать после получения образца файла).
6.2 Файл 2 — наличие и цены: обновление остатков и цен по ключу сопоставления с каталогом (артикул/SKU — уточнить по файлу).
6.3 Журнал загрузок: кто, когда, файл, результат (успех/ошибки по строкам), возможность отката — по согласованию.
6.4 Ручное редактирование единичных позиций в админке — опционально, обсудить с заказчиком.


7. Дизайн и UX
7.1 Простой, «воздушный» интерфейс без перегруза декора: понятная навигация, читаемая типографика, адаптивная вёрстка для мобильных устройств.
7.2 Доступность базового уровня: контраст, подписи к полям форм, понятные сообщения об ошибках.


8. Персональные данные и правовые требования
8.1 Публикация на сайте: Политика конфиденциальности, Согласие на обработку персональных данных при регистрации/оформлении заказа (чекбоксы с хранением факта и версии текста), при необходимости — Cookies/файлы cookie и аналитика (если планируется).
8.2 Минимизация данных: собирать только то, что нужно для заказа и связи; ограничение доступа в админке; журналирование действий администратора с персональными данными (просмотр заказа и т.д.) — по согласованию с юристом заказчика.
8.3 Технические меры: HTTPS, безопасное хранение паролей (хэш), защита от перебора при входе и сбросе пароля, резервное копирование, обновления зависимостей — в объёме, согласованном с исполнителем.
8.4 Юрисдикция и применимое законодательство (например 152-ФЗ при работе с данными граждан РФ) — уточнить у заказчика и отразить в юридических документах.
